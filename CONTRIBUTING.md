# Contributing

Before non-trivial work, read [AGENTS.md](AGENTS.md) and the complete
[operating rules](docs/it/00-regole-operative.md).

Changes should be small, linked to an issue, covered by deterministic tests, and documented.
Public APIs require docstrings and Sphinx reference coverage. Rendering changes require readable
ASCII snapshots, narrow-terminal cases, and ANSI/`no-color` checks.

Run the verification commands documented in the operating rules before opening a pull request.
Use the repository PR review skill for the two required clean review rounds.
